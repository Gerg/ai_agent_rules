## AGENTS.md

Learn more about the spec: https://agents.md/

I use AGENTS.md in a non-standard way. The intended use case is akin to a
README.md file, namely rules that are tied to the particular code repository
and checked in alongside source code.

Instead, I maintain a global AGENTS.md file with generic rules, symlink it into
each individual repository, and do not check it in. This has a number of
weaknesses, including:
1. It's tedious to maintain the symlinks
1. It conflicts with any repo that already has an AGENTS.md file

Ideally, I'd want something like an `~/AGENTS.md`, which automatically applies
globally, but I'm not aware of anything like that. The closest is probably
Cursor user rules, but they only apply to Cursor and are configured via the
Cursor IDE, rather than a configuration file that can be easily edited, checked
into a VCS, etc.
